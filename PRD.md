## 1. 제품 개요

### 1.1 한 줄 정의
**씨젠 의료사업부 이무팀이 매주 작성하는 주간업무 보고서**를 칸반식 업무관리 + Word형 인라인 편집기 + 자동 차주 인계 기능으로 통합한 **단일 HTML 파일 웹앱**.

### 1.2 사용 환경
- 데스크탑 크롬/엣지 권장, 화면폭 1280px 이상.
- 파일 더블클릭으로 실행 (file:// 또는 사내 공유 폴더).
- 저장소: `localStorage` 키 `ga_erp_v1`.

### 1.3 기술 스택
- **HTML5 + CSS3 (custom properties)** — 외부 CSS 프레임워크 없음.
- **Vanilla JavaScript (ES2017+)** — 빌드 도구 없음, 단일 `<script>` 블록.
- **Google Fonts**: Noto Sans KR (앱 UI), 맑은 고딕 (Word 출력).
- **contenteditable + document.execCommand** — 리치텍스트 편집.
- **Blob + data URI** — Word(.doc) 및 JSON 다운로드.

## 2. 정보 구조 (IA) — 4개 탭

```
[헤더] 씨젠  로고 → 이무팀 → 🏠대시보드 → 📋업무현황 → 📄주간보고서 → ⚙설정 → 오늘날 짜 → 주차 → +업무추가
```

### Tab 1. 🏠 대시보드
- **상단**: 4이 칸반 보드 (🚨긴급 / 📅예정 / 🔄진행중 / ✅완료) — 본 주차 업무 카드.
- **하단**: 금주 업무 보고 요약 — 팀원별로 "구분 / 금주 업무내용 / 기간"만 발췌한 표.
- 빈 상태일 때 "업무 추가하기" / "주간보고서 작성하기" CTA 노출.

### Tab 2. 📋 업무현황
- 필터 바: 담당자 / 상태(세그먼트드 버튼) / 검색.
- 간이 칸반 4이 구조. 상태 필터 선택 시 단일 열 1fr로 펼침.
- 카드 클릭 시 ✏ 수정 / 🗑 삭제. + 헤더의 "+업무추가" 버튼으로 모달 열기.

### Tab 3. 📄 주간보고서 (핵심)
- **상단 컨트롤 바**
  - 주차 내비게이션 (‹ › / 이번주 / 라벨 클릭 → 달력 표시)
  - 팀원 탭 (전체 / 팀원별 pill)
  - 우측 컨트롤: 📄 Word 추출 / 📥 지난주 업무 불러오기 / 💾 저장 / 개인·전체 토글 / 🔍 기간 조회
- **서식 툴바 (sticky)**
  - **B U** / 색상(빨/파/검) / ☰ 표 (8×8 그리드 피커) / 🖼 이미지 / ✕ 서식제거
- **문서 본문**
  - 헤더: "씨젠 의료사업부 이무팀 주간업무 일지" + 작성일/작성자/부서 + 주차 라벨.
  - 팀원별 섹션 → 8열 Word형 테이블 (수정 후 추가/삭제):
    `구분 / 금주 업무내용(rich) / 일정 / 담당자 / 차주 업무내용(rich) / 일정 / 담당자 / 비고`
  - "＋ 구분 추가" 대시 버튼으로 행 추가.
  - 행 내부 우클릭 → 컨텍스트 메뉴 (위/아래 행 추가, 좌/우 열 추가, 행/열 삭제, 셀 병합·해제, 표 삭제).
- **자동 인계 모달** — 현 주차에 데이터가 0개일 때 직전 주차 차주 계획 import 안내.

### Tab 4. ⚙ 설정
- 팀원 관리: 이름 + 직급 입력 → 추가/삭제. 직급은 인라인 input으로 수정.
- 업무 분류 관리: 분류명 추가/삭제 (기본값: 행정·구매·센터/지원·대외·업무지시·시설관리·보고/기타·기타).
- 데이터 관리: 📤 JSON 내보내기 / 📥 JSON 불러오기 / 🗑️ 업무 데이터 전체 초기화 (팀원·분류는 유지).
- 앱 정보: 앱명·버전 (v1.2.0)·저장방식 표시.

## 5. 핵심 사용자 플로우

### 5.1 신규 업무 추가
1. 헤더 `+ 업무추가` → 모달.
2. 담당자(복수 선택 가능, 체크박스 dropdown), 분류(기존 선택 or 직접 입력), 업무명, 세부항목(줄바꿈 = 다지열우기), 상태, 시작일/마감일(미정 체크 가능), 비고 입력.
3. 저장 시 `week`는 자동으로 현재 활성 주차로 기록.

### 5.2 주간보고서 작성·저장
1. 주간보고서 탭 진입 → `RS.init()`이 현재 주차 데이터 로드.
2. "＋ 구분 추가"로 행 생성 → 직접 입력 (oninput → `RS._markDirty()`).
3. 💾 저장 (`Ctrl+S` 단축키 가능) → `RS._syncDOM()` 으로 DOM → state 동기화 후 localStorage 반영.
4. 미저장 상태에서 주차 이동·기간 모드 진입 시 confirm 경고.

### 5.3 차주 → 금주 자동 인계
1. 신 주차 진입 시 현재 주차 데이터 0개 + 이전 주차 `nextContent` 존재 → 자동 모달.
2. 체크박스 선택 → `RS.importSuggestions()` 호출 → 이전 주차의 `nextContent/Date/Person`을 선택한 `thisContent/Date/Person`으로 복사.
3. 한 번 닫으면 `_dismissedWeek` 기록되어 같은 주차에 다시 안 뜸.

### 5.4 Word 추출
- **단일 주차**: `RS.exportWord()` → `씨젠 이무팀_주간업무_2026-W20.doc` (개인 보기면 해당 팀원만, 전체면 모든 팀원 섹션).
- **기간**: 기간 조회 모드에서 `exportWordPeriod()` → `씨젠 이무팀_주간업무_{이름}_{N}주치.doc`.
- 공통: `_buildWordDoc()` 골격, 가로 A4(297×210mm), 씨젠  상단 헤더, 작성일 자동 삽입, contenteditable 속성 정리.

### 5.5 데이터 동기화
- 한 팀원이 작성 후 ⚙ 설정 → 📤 JSON 내보내기 → 파일 공유.
- 다른 팀원이 같은 화면에서 📥 JSON 불러오기 → `Store.save(d)` → `App.refresh()`.

## 7. 알려진 버그·주의 사항 (차기 작업에서 인지)

1. **이미지 삽입 시 500KB 초과 경고** — base64로 localStorage에 저장되므로 5MB 한도 주의.
2. **`document.execCommand`는 deprecated** — 현재 모든 리치텍스트 처리가 이에 의존. 향후 ContentEditable API 마이그레이션 필요할 수 있음.
3. **`memberInfo` 키 불일치 케이스 존재** — 샘플 JSON에서 `"김성일 팀장"`(공백 포함)과 `"김성일"`이 별도 key로 존재. `addMember(name, title)` 사용 시 일관성 유지 필요.
4. **`members` 항목의 직급 포함 여부 혼재** — 일부 데이터는 `"김성일 팀장"`처럼 이름+직급 결합형. memberInfo.title 분리 이용 권장.
5. **차주 인계 모달은 "데이터 0개일 때만" 자동** — 행 하나라도 있으면 자동 안 뜸. 수동 import는 📥 버튼으로.
6. **`isoWeek` 연산은 ISO 8601** — 목요일이 속한 해의 주차. 1월 1일이 일/월요일이면 전년도 W52/W53으로 잡힐 수 있음.
7. **모든 escape는 `escH()` 사용** — `innerHTML` 직접 주입 시 누락 주의 (특히 사용자 입력 텍스트).
8. **`reportData[week][person]`은 팀원 이름 키** — 팀원 이름 변경 시 reportData 마이그레이션 별도 필요(현재 미구현).
